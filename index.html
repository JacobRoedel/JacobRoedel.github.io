Mini Search Engine

Mini Search Engine is a document retrieval system made in C. I created three unique structs to maintain a hashmap of linked lists, in which each node in the first layer of linked lists was the head of a second layer of linked lists. The first layer of linked lists contain nodes which represent each unique word that appears in all of the documents we are searching through, and the total number of documents it appears in. The second layer of linked lists represents each document that the word appears, and how many times it appears in a given document.

Training Phase
The first phase of this project is taking each unique word that appears in the total set of documents, and adding it to the hashmap structure. After, I remove stop words, words that appear frequently such as articles, so that it doesn't affect the ranking of the documents.

Search and Ranking Phase
The second phase of this project is to use the tf-idf ranking system (term frequency-inverse document frequency) to determine the relevancy of a given word. This ranking is determined by multiplying the frequency of a word in the given document and the inverse document frequency (log(Document Frequency/# of Documents)). I then summed the tf-idf for each word in each document, and used a bubble sort to display the ranking in descending order of the set of documents.

Encyrption

Encyrption is a basic encryption project in LC-3 assembly language that incorporates Vigenere and Caeser ciphers to both encrypt and decrypt a 10 character input wtih a 5 character input string. I implemented subroutines to implement multiplication, modulo, XOR and other various functions. I stored the ASCII values of the encrypted/decrypted input in memory at seperate locations, and cleared the memory upon request to exit the program

Cat and Mouse Simulation

Cat and Mouse Simulation is a OOP based project in Java that plots creatures onto a visual grid. A hashmap is used to represent the entire grid, and linked lists at each coordinate point to store a list of creatures at each point. There are 4 unique types of creatures in the simulation - Mouse, Cat, Zombie Cat and Terminator.

Mice
- Randomly turns direction 20% of the time, and moves one grid point per round
- Procreates every 20 rounds, meaning a new mouse spawns at the current point of the parent mouse
- Dies after 30 rounds of simulation

Cat
- Randomly turns direction 5% of the time, and moves two grid points per round
- Eats mice if they both land on the same grid point
- Searches/Chases mice within a 20 grid point radius
- Transforms to a zombie cat if it has not eaten in 50 rounds

Zombie Cat
- Mimics the movment of a cat when not chasing, and moves three grid points per round
- Eats mice and cats if they land on the same grid point: cats turn into zombie cats when eaten
- Searches/Chases mice and cats within a 40 grid point radius
- Dies after 100 rounds without eating

Terminator
- Randomly turns direction every round
- Eats mice, cats, and zombie cats
- Does not track/chase any creatures, and is made to counter the zombie cats

Simulation
- A new mice spawns every 100 rounds
- A new cat spawns every 25 rounds
- A new terminator spawns every 100 rounds
